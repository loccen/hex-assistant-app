# 发布、运行、诊断和验收说明

## 1. 文档定位

本文补齐正式项目的发布、运行、诊断和验收说明。它面向 `/home/code/hex-assistant-app`，不适用于旧 POC 仓库 `/home/code/hex-assistant`。

本文只描述应如何发布和验收，不把尚未真实执行的 Windows 局内测试写成已通过。当前 WSL 环境可以做文档检查、前端构建和部分静态检查，但不能真实验收 Windows Overlay、点击穿透、LOL 截图、OCR 局内闭环和完整端到端流程。

## 2. 运行和打包命令

所有安装、构建、开发和打包命令必须在项目根目录通过 `mise exec` 执行。

基础验证：

```bash
mise trust
mise install
mise exec -- npm install
mise exec -- npm run build
mise exec -- npm run check:rust
```

开发运行：

```bash
mise exec -- npm run tauri dev
```

Windows 发布打包：

```bash
mise exec -- npm run tauri build
```

打包产物路径以 Tauri 构建日志为准，通常在：

```text
src-tauri/target/release/
src-tauri/target/release/bundle/
```

文档改动至少执行：

```bash
git diff --check
```

## 3. Windows 验收条件

以下内容必须在真实 Windows 桌面环境验收，不能由 WSL 代替：

- LOL 无边框模式下的显示器级截图。
- 黑屏和旧帧检测。
- 用户校准区域是否匹配实际三张海克斯卡片。
- PP-OCRv4 rec ONNX 模型在发布包中加载。
- `onnxruntime.dll` 在无开发环境变量时加载。
- Overlay 透明、置顶、不抢焦点和点击穿透。
- 多显示器、Windows 缩放、游戏 UI 缩放下的坐标一致性。
- Live Client 状态机在真实局内 3 / 7 / 11 / 15 级流程中的迁移。
- ApexLOL 成功、缓存、失败和无数据兜底。

如未完成上述测试，验证记录必须写“未验收”，不能写“通过”。

## 4. 发布包清单

发布包应包含：

- Windows 可执行文件或安装包，例如 `*.exe`、`*.msi` 或 `*setup.exe`。
- PP-OCRv4 rec ONNX 模型和必要的字典 / 元数据。
- ONNX Runtime Windows 动态库，例如 `onnxruntime.dll`。
- 运行说明。
- 已知限制。
- 诊断包导出说明。
- 验证记录。
- 产物校验值。

建议发布目录：

```text
release/
  README.md
  验证记录.md
  packages/
  models/
  runtime/
  docs/
  diagnostics/
```

发布前必须在干净 Windows 环境确认：应用不依赖源码目录、pip 包目录、`ORT_DYLIB_PATH` 或开发机上的临时资源路径。

## 5. 运行说明

推荐运行路径：

1. 在 Windows 桌面启动应用。
2. 将 LOL 设置为无边框模式。
3. 进入海克斯选择界面。
4. 选择游戏所在显示器。
5. 完成三张名称区域、三张底部锚点和底部展示 / 隐藏按钮区域校准。
6. 保存校准配置。
7. 进入正常监听。
8. 到达 3 / 7 / 11 / 15 级时，助手识别三张海克斯并展示 ApexLOL 来源信息。

异常时应进入可解释状态：

- 截图黑屏或疑似旧帧：暂停识别，提示切换无边框或重新截图。
- 未校准：不展示 Overlay，提示先校准。
- OCR 低置信度：提示疑似结果或要求人工确认，不直接查询 ApexLOL。
- ApexLOL 失败：显示“暂无数据”，记录失败原因。
- Overlay 异常：隐藏 Overlay，提示重新校准或重启。

## 6. 已知限制

- 独占全屏不是 MVP 默认支持场景。即使截图不是黑屏，也必须结合旧帧检测和连续样本证明可用。
- WSL 不能验收 Windows Overlay、点击穿透、真实截图和 LOL 局内状态。
- 多显示器和缩放环境需要分别验收，不能用单屏成功结论覆盖全部场景。
- OCR 成功依赖校准区域、模型、ORT 动态库和图像质量，任何一项异常都要保留诊断材料。
- ApexLOL 是外部页面来源，页面结构变化或网络失败必须显示兜底状态。
- 应用不自动点击、不自动选择、不自动重随、不读取游戏内存、不注入或 Hook。

## 7. 诊断包导出

诊断包用于让开发者复盘问题。建议导出为：

```text
diagnostics-YYYYMMDD-HHMMSS/
  manifest.json
  logs/
  capture/
  ocr/
  overlay/
  apexlol/
  validation/
```

`manifest.json` 应包含：

- 应用版本、提交号、构建时间。
- Windows 版本、显示器编号、分辨率、缩放。
- LOL 显示模式、地图、游戏 UI 缩放。
- OCR 模型路径、ORT 动态库路径和加载状态。
- ApexLOL 查询状态。
- 导出时间、测试人和问题描述。

`logs/` 应包含：

- 应用主日志。
- 状态机日志。
- Overlay 日志。
- ApexLOL 查询日志。

`capture/` 应包含：

- 原始截图。
- 截图元数据。
- 黑屏检测结果。
- 旧帧检测结果。
- 三个 slot 的裁剪图。

`ocr/` 应包含：

- OCR 原始文本。
- 置信度。
- 匹配分。
- 最终标准名称。
- 增强图。
- 低置信度或失败原因。

`overlay/` 应包含：

- Overlay 窗口坐标。
- 三张说明卡片坐标和尺寸。
- 点击穿透测试记录。
- 截图或录屏说明。

`apexlol/` 应包含：

- 请求 URL。
- HTTP 状态和耗时。
- 解析结果。
- 缓存命中状态。
- 失败或无数据原因。

诊断包不得保存敏感令牌，不得保存与本功能无关的个人信息。

## 8. 问题定位指南

### 8.1 截图问题

优先检查：

- 是否在 Windows 桌面环境运行。
- LOL 是否为无边框模式。
- 目标显示器编号是否正确。
- 原始截图是否黑屏。
- 连续截图是否为旧帧。
- 分辨率、Windows 缩放、游戏 UI 缩放是否和校准时一致。

需要留存：

- 原始 PNG。
- `capture-meta.json`。
- 黑屏检测结果。
- 旧帧哈希或相似度结果。
- 目标显示器信息。

### 8.2 OCR 问题

优先检查：

- 三个名称区域裁剪图是否准确。
- 增强图是否能看清文字。
- OCR 模型是否来自发布包资源目录。
- `onnxruntime.dll` 是否加载成功。
- raw 文本、置信度和匹配分是否合理。
- 低置信度结果是否被错误用于查询。

需要留存：

- 三个 slot 裁剪图。
- 增强图。
- OCR 结果 JSON。
- 模型加载日志。
- ORT 动态库加载日志。

### 8.3 ApexLOL 问题

优先检查：

- 英雄名称是否来自 Live Client。
- 海克斯名称是否为 OCR 匹配后的标准名称。
- 请求 URL 是否正确。
- 页面是否可访问。
- 解析失败时是否显示“暂无数据”。
- 缓存键是否按“英雄 + 海克斯”区分。

需要留存：

- 请求 URL。
- HTTP 状态和耗时。
- 解析摘要。
- 缓存记录。
- 成功、缓存、失败、无数据四类展示样本。

### 8.4 Overlay 问题

优先检查：

- Overlay 是否创建在正确显示器。
- 窗口是否透明、置顶、不抢焦点。
- Windows 点击穿透是否真实生效。
- 三张说明卡片是否根据底部锚点定位。
- 是否遮挡选择按钮、重随按钮、名称或底部按钮。
- 面板收起时是否隐藏，展开后是否恢复。

需要留存：

- Overlay 坐标 JSON。
- 卡片坐标和尺寸日志。
- 展示截图或录屏。
- 点击穿透测试记录。
- 多显示器信息。

### 8.5 状态机问题

优先检查：

- Live Client API 是否可用。
- 当前英雄、等级和时间戳是否正确。
- 3 / 7 / 11 / 15 级待处理队列是否生成。
- 面板收起是否被错误标记为完成。
- 完成候选是否经过短时间复核。
- 重随后是否只刷新变化 slot。
- 异常状态是否暂停自动展示。

需要留存：

- 状态迁移日志。
- Live Client 返回摘要。
- 待处理队列日志。
- 面板展开 / 收起样本。
- 重随前后 slot 对比。
- 异常恢复记录。

## 9. 验收记录规则

发布前必须更新 `release/验证记录.md`。记录规则：

- 只写实际执行过的命令和测试结果。
- 未执行项目写“未验收”。
- 有问题但不阻断发布时写“带问题通过”，并说明影响范围。
- WSL 结果只能证明对应的 WSL 命令结果，不能证明 Windows 局内功能通过。
- 旧 POC 结论只能作为风险背景，不能作为正式项目通过证据。

每条验收记录至少包含：

- 测试时间。
- 测试人。
- 系统环境。
- 显示模式。
- 分辨率和缩放。
- 输入样本或操作步骤。
- 期望结果。
- 实际结果。
- 证据路径。
- 结论。
