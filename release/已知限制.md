# 已知限制

- 当前 WSL 环境可以完成前端构建、Rust 静态检查、Rust 单元测试和 Linux Tauri 产物尝试，不能验收 Windows Overlay、点击穿透、真实 LOL 截图或局内端到端流程。
- Windows `exe` / `msi` 需要在 Windows runner 上构建，或准备完整 Tauri 交叉编译条件后另行验证。
- 仓库已随包包含真实 `ppocrv4_rec.onnx`、Windows x64 ONNX Runtime 1.25.0 `onnxruntime.dll` 和包内 `onnxruntime_providers_shared.dll`，但这些资源在干净 Windows 桌面环境中的实际加载仍未验收。
- 独占全屏不是 MVP 默认支持场景。推荐 LOL 使用无边框模式。
- 多显示器、Windows 缩放、游戏 UI 缩放和分辨率变化都会影响截图裁剪、OCR 与 Overlay 坐标，必须分别验收。
- ApexLOL 页面结构变化、断网或无数据时应显示“暂无数据”，不能伪造推荐结果。
- 助手只展示信息，不自动点击、不自动选择、不自动重随、不读取游戏内存、不注入或 Hook 游戏进程。
